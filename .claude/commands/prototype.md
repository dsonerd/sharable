Build a quick prototype or spike solution for the given idea.

## Instructions

1. Clarify the goal and success criteria before coding.
2. Keep it minimal — focus on proving the concept, not production quality.
3. Include a brief README or header comment explaining what the prototype does and how to run it.
4. Save the result to `prototypes/<topic-in-kebab-case>/` as a self-contained folder.
5. If a prototype on the same topic already exists, update it rather than creating a new one.
6. Commit the result when done.
