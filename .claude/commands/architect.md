# Architect

Design system architecture for the given topic or component.

## Instructions

1. Clarify scope and constraints before designing.
2. Produce a structured architecture document covering:
   - Context and problem statement
   - Design goals and constraints
   - High-level architecture (components, interactions, data flow)
   - Key decisions and trade-offs (ADR style where appropriate)
   - Diagrams described in text or Mermaid syntax
3. Save the result to `architecture/<topic-in-kebab-case>.md` (use a subfolder if the design has multiple documents).
4. If an architecture document on the same topic already exists, update it rather than creating a new one.
5. Commit the result when done.
